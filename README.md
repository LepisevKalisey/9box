Документация приложения 9Box Talent Matrix
1. Обзор приложения
9Box Talent Matrix — это веб-приложение для оценки кадрового потенциала компании по методике "9 ячеек" (9-box grid). Приложение позволяет руководителям оценивать сотрудников по двум осям: Результативность (Performance) и Потенциал (Potential), используя структурированный опросник.
Ключевые возможности:
Многопользовательская система: Разделение ролей на Администратора и Менеджеров.
Изоляция данных: Менеджеры видят сотрудников только своей компании.
Опросник: Встроенная методика оценки с балльной системой.
Визуализация: Интерактивная матрица 9x9 и списочное представление.
ИИ-Рекомендации: Интеграция с Google Gemini API для генерации индивидуальных планов развития сотрудников.
Адаптивность: Полная поддержка мобильных устройств.
2. Технический стек
Frontend (Клиентская часть)
React 18: Библиотека интерфейса.
TypeScript: Строгая типизация.
Tailwind CSS: Стилизация компонентов.
Vite: Сборщик проекта.
Lucide React: Иконки.
Google GenAI SDK: Взаимодействие с нейросетью.
Backend (Серверная часть)

3. Установка и запуск
Вариант А: Запуск через Docker Compose (Рекомендуемый, совместим с Coolify)
Этот способ автоматически собирает фронтенд, поднимает сервер и настраивает постоянное хранилище данных.
Требования: Установленный Docker и Docker Compose (или платформа Coolify).
Настройка API ключа (опционально):
Перед запуском задайте переменную окружения API_KEY (ключ Google Gemini). В Coolify укажите её в Environment Variables.
Файл docker-compose.yml уже включён в проект. Пример:
code
Yaml
version: '3.8'
services:
  app:
    build: .
    image: 9box-talent-matrix:latest
    environment:
      - NODE_ENV=production
      - PORT=3001
      - API_KEY=${API_KEY}
    ports:
      - "3001:3001"
    volumes:
      - 9box_data:/app/data
    restart: unless-stopped
volumes:
  9box_data:
    driver: local
Запуск:
В корне проекта выполните команду:
code
Bash
docker compose up --build -d
Доступ:
Приложение будет доступно по адресу: http://localhost:3001
Данные сохраняются в Docker Volume 9box_data, поэтому они не исчезнут при перезапуске контейнера.
Вариант Б: Локальный запуск (для разработки)
Установка зависимостей:
code
Bash
npm install
Запуск сервера:
code
Bash
node server.js
(Сервер запустится на порту 3001).
Запуск фронтенда (в другом терминале):
code
Bash
npm run dev
(Vite запустит сервер разработки, обычно на порту 5173, с проксированием запросов на API).
4. Руководство Администратора
Вход в систему
По умолчанию создается учетная запись супер-администратора:
Email: admin@mides.kz
Пароль: 128500
Функционал Панели Администратора
В кабинете администратора доступны 4 вкладки:
Общая матрица:
Просмотр распределения всех сотрудников по ячейкам.
Фильтры: Можно отфильтровать результаты по конкретной Компании или конкретному Менеджеру.
В этом режиме администратор может нажать на сотрудника и сгенерировать ИИ-план развития.
Компании:
Создание новых компаний.
Удаление компаний.
Внимание: При удалении компании удаляются ВСЕ связанные с ней менеджеры, сотрудники и оценки.
Менеджеры:
Создание учетных записей для руководителей.
При создании менеджера необходимо выбрать Компанию, к которой он принадлежит.
Менеджеру присваивается Email (логин), Имя и Пароль.
База сотрудников:
Полный список всех сотрудников, заведенных в систему всеми менеджерами.
Возможность удаления сотрудника (удаляет также историю его оценок).
5. Руководство Пользователя (Менеджера)
Процесс оценки
Выбор сотрудника:
Менеджер может выбрать сотрудника из списка (если он уже был создан ранее в рамках компании).
Или создать нового сотрудника ("Новый сотрудник"), указав ФИО и Должность.
Оценка (Анкетирование):
Процесс состоит из последовательных вопросов.
Блок 1: Результативность (Performance): Качество, Автономность, Эффективность.
Блок 2: Потенциал (Potential): Обучаемость, Масштаб мышления, Амбиции.
Блок 3: Калибровка: Риск ухода, Готовность к повышению.
Ответы автоматически сохраняются при завершении опроса.
Результаты:
После оценки менеджер видит матрицу 9-box:
Grid View (Сетка): Классическая матрица (удобно на ПК).
List View (Список): Группировка по категориям (удобно на мобильных).
Детали и ИИ:
Клик по сотруднику открывает детальную карточку.
Кнопка "Создать план" отправляет данные (ФИО, Должность, Текущий квадрат матрицы) в Gemini API.
ИИ возвращает рекомендации: Диагноз, План действий и Риски.
6. Методика расчета
Приложение использует следующий алгоритм для определения позиции в матрице:
Каждый вопрос имеет вес от 0 до 2 баллов.
Суммируются баллы отдельно по категории Performance и Potential.
Сумма переводится в уровень:
0-1 балл: Low (Низкий)
2-4 балла: Moderate (Средний)
5+ баллов: High (Высокий)
На пересечении уровней определяется итоговый квадрат (например, High Potential + Low Performance = "Темная лошадка").
7. Структура API (Backend)
Сервер (server.js) предоставляет REST API:
POST /api/login - Аутентификация.
GET /api/companies - Список компаний.
POST /api/companies - Создание компании.
DELETE /api/companies/:id - Удаление компании (каскадное).
GET /api/users - Список пользователей.
POST /api/users - Создание пользователя (только админ).
DELETE /api/users/:id - Удаление пользователя.
GET /api/employees - Список сотрудников.
POST /api/employees - Создание сотрудника.
GET /api/assessments - Получение всех оценок.
POST /api/assessments - Сохранение оценки.
8. Безопасность и Ограничения
Авторизация: Простая проверка по файлу JSON. В текущей версии нет токенов (JWT) или сессий на стороне сервера. Сессия хранится в localStorage браузера.
Хранение данных: Используется файл db.json. При высокой нагрузке рекомендуется миграция на PostgreSQL или MongoDB (потребует переписывания server.js).
Изоляция: Сервер проверяет роль пользователя (admin или manager) перед выполнением критических операций (создание/удаление пользователей).
